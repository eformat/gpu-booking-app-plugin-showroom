= GPU Booking Plugin Workshop
:navtitle: Welcome
:page-layout: default

Welcome to the GPU Booking Plugin Workshop. In this hands-on workshop, you will learn how to deploy, configure, and use the GPU Booking Plugin for OpenShift — a complete solution for managing GPU resource reservations with Kueue integration.

== What You Will Learn

By the end of this workshop, you will be able to:

* Understand the GPU Booking Plugin architecture (Go backend + React/PatternFly frontend)
* Deploy the plugin to OpenShift using Helm
* Create and manage GPU bookings through the OpenShift Console
* Understand Kueue integration for automatic workload tracking
* Administer the booking system including quota management

== Who This Is For

This workshop is designed for:

* **Platform Engineers** responsible for GPU infrastructure on OpenShift
* **ML Engineers** who need to reserve GPU resources for training and inference workloads
* **Cluster Administrators** managing multi-tenant GPU clusters
* **Developers** building OpenShift Console plugins

== Prerequisites

Before starting this workshop, you should have:

* Basic familiarity with OpenShift and kubectl/oc commands
* Understanding of Kubernetes concepts (Pods, Services, ConfigMaps)
* A cluster with NVIDIA GPUs and MIG support configured
* OpenShift 4.14+ with the console enabled

== Estimated Time

This workshop takes approximately **2 hours** to complete.
